# Web Patterns

## API Client

- Auto-generated from OpenAPI spec in `src/autogenerated/` -- never edit manually.
- Regenerate after backend changes: `pnpm generate:api`
- Import types from `src/autogenerated/api`.
- Use Refine hooks (`useList`, `useOne`, `useCreate`, `useUpdate`, `useDelete`) for standard CRUD.

## Components

- Functional components only. No class components.
- Use `React.ComponentProps<'element'>` for extending HTML element props.
- Destructure props in the function signature.
- Avoid large components; extract sub-components or custom hooks when a component exceeds ~200 lines.

## Hooks

- Custom hooks go in `src/hooks/` for shared hooks, or co-located with the feature for feature-specific hooks.
- Use `useRef` for values that need to be current in callbacks without causing re-renders (handler wrapping pattern).
- Always clean up subscriptions/listeners on unmount via `useEffect` return.
- WebSocket: use `useWebSocket`, `useGraphWebSocket`, or `useThreadWebSocket` hooks. Never subscribe to WebSocketService directly in components.

## State Management

- Refine core for server state (data fetching, caching, mutations).
- React Context for shared client state that multiple components need.
- `localStorage` via `GraphStorageService` for canvas viewport persistence.
- No Zustand, Jotai, or other state libraries.

## Types

- Feature-specific types go in `.types.ts` files per feature/directory.
- Re-export API types from autogenerated client when needed; don't duplicate them.
- Use `interface` for object shapes, `type` for unions/intersections/utility types.

## Services

- Singleton instances or static utility classes in `src/services/`.
- `WebSocketService` for socket management, `GraphStorageService` for local persistence, `validationService` for graph validation.

## Styling

- Tailwind utility classes exclusively. No inline `style` props unless dynamic values require it.
- `cn()` from `@/components/ui/utils` for conditional class merging.
- `cva()` from `class-variance-authority` for component variants.
- Component primitives from Radix UI, styled with Tailwind + CVA in `src/components/ui/`.
- No CSS modules, no styled-components, no CSS-in-JS, no Ant Design.

> **Note:** `claude.md` and `CLAUDE.md` reference Ant Design, but it is not installed. The codebase uses Tailwind + CVA + Radix exclusively. Ignore Ant Design references in those files.

## Validation

```bash
pnpm run full-check    # build + lint (mandatory before finishing)
```
